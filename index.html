Beginner's Quick Start
A simple step-by-step guide to get your first project onto Git and GitHub
1. Install Git
First, you need to have Git on your machine. Go to the Installation & GUIs section above to download it. Git is the engine that tracks changes to your code.
2. Add and Commit (What's the difference?)
Why do we use two commands to save?
  • Add (git add .): This is the staging area. Think of it like putting items into a shopping cart. You're selecting which changed files you want to include in your next save.
  • Commit (git commit -m "my message"): This is like checking out. It permanently takes a "snapshot" of the items in your cart and saves it in your project's history along with a descriptive message.
3. Push for the First Time
Now that you have a committed snapshot locally, you need to send it to GitHub so others can see it (or to back it up). For your first push to a new remote repository, run:

git remote add origin <your-repo-url>
git push -u origin main

This uploads your code and links your local branch to the remote one.
4. Next Steps
To continue your work, simply repeat the cycle:
  • 1. Make changes to your files and save them.
  • 2. Run git add . to stage the new changes.
  • 3. Run git commit -m "described what I changed" to save the snapshot.
  • 4. Run git push to upload your snapshot to the remote repo.
5. Getting Updates
What if someone else pushed something to the repository?

Before you start making new changes, you should always make sure your computer has the latest version of the code. To download and merge their changes with your local files, run:
git pull

It is best practice to pull frequently to avoid having conflicting changes!

Setup
Configure your Git identity and global preferences
Identity
Set Username
Set the name that will be attached to all your commits.
git config --global user.name "Your Name"
Identity
Set Email
Set the email address that will be attached to all your commits.
git config --global user.email "you@example.com"
Preferences
Set Default Editor
Choose the editor Git opens for commit messages (e.g. VS Code, Vim).
git config --global core.editor "code --wait"
Preferences
Set Default Branch Name
Change the name of the initial branch from master to main.
git config --global init.defaultBranch main
Review
List All Config
Show all global Git configuration values currently in effect.
git config --global --list
Preferences
Enable Color Output
Turn on colorized output for easier reading in the terminal.
git config --global color.ui auto

Setup & Init
Initialize new repositories or clone existing ones
Initialize
Create New Repo
Initialize a new Git repository in the current directory. Creates a hidden .git folder.
git init
Initialize
Init in Named Folder
Create a new directory and initialize a Git repository inside it.
git init my-project
Clone
Clone a Repository
Copy an existing remote repository to your local machine, including full history.
git clone https://github.com/user/repo.git
Clone
Clone into Folder
Clone a repository into a specific local folder name.
git clone https://github.com/user/repo.git my-folder
Clone
Shallow Clone
Clone only the most recent commit — much faster for large repositories.
git clone --depth 1 https://github.com/user/repo.git

Stage & Snapshot
Track changes and create commits in your repository
Status
Show Working Status
Show the state of modified, staged, and untracked files in your working directory.
git status
Stage
Stage a File
Add a specific file to the staging area so it will be included in the next commit.
git add <filename>
Stage
Stage All Changes
Stage all modified and new files in the current directory and subdirectories.
git add .
Stage
Stage Interactively
Interactively choose which hunks (portions) of a file to stage — great for partial commits.
git add -p <filename>
Diff
Show Unstaged Changes
View the line-by-line diff of what changed in the working directory but hasn't been staged yet.
git diff
Diff
Show Staged Changes
View the diff of what is staged and will go into the next commit.
git diff --staged
Commit
Commit Staged Changes
Record staged changes into the repository history with a descriptive message.
git commit -m "Your commit message"
Commit
Stage & Commit
Automatically stage all tracked modified files and commit in one step.
git commit -am "Your commit message"
Unstage
Unstage a File
Remove a file from the staging area while keeping your working directory changes.
git restore --staged <filename>
Discard
Discard Working Changes
Discard all unstaged changes in a file, reverting it to the last committed state.
git restore <filename>

Branch & Merge
Isolate work and integrate changes between branches
List
List Branches
Show all local branches. The current branch is highlighted with an asterisk.
git branch
List
List All Branches
Show local and remote-tracking branches together.
git branch -a
Create
Create a Branch
Create a new branch pointing at the current commit without switching to it.
git branch <branch-name>
Switch
Switch to Branch
Switch your working directory to another branch.
git switch <branch-name>
Create & Switch
Create & Checkout Branch
Create a new branch and immediately switch to it in one command.
git switch -c <branch-name>
Merge
Merge a Branch
Merge the specified branch into the current branch, creating a merge commit if needed.
git merge <branch-name>
Merge
Merge — No Fast Forward
Force a merge commit even when a fast-forward is possible, preserving branch history.
git merge --no-ff <branch-name>
Delete
Delete a Branch
Delete a branch that has already been merged. Use -D to force-delete unmerged branches.
git branch -d <branch-name>

Inspect & Compare
Examine logs, diffs, and repository history
Log
Show Commit Log
Display the full commit history with author, date, and message for the current branch.
git log
Log
Compact Log
Show an abbreviated one-line-per-commit history — great for a quick overview.
git log --oneline --graph --all
Log
Log by Author
Filter commit history to show only commits from a specific author.
git log --author="Name"
Diff
Diff Between Branches
Show what has changed between two branches.
git diff main..feature-branch
Show
Show a Commit
Display the diff and metadata for a specific commit by its hash.
git show <commit-hash>
Blame
Annotate File
Show which commit and author last modified each line of a file.
git blame <filename>

Share & Update
Sync with remote repositories and collaborators
Remote
Add Remote
Connect your local repository to a remote server like GitHub.
git remote add origin https://github.com/user/repo.git
Remote
List Remotes
Show all remote connections configured for the current repository.
git remote -v
Push
Push a Branch
Upload local commits on a branch to the corresponding remote branch.
git push origin <branch-name>
Push
Push & Set Upstream
Push the branch and configure it to track the remote branch for future git push/pull.
git push -u origin <branch-name>
Fetch
Fetch Remote Changes
Download objects and refs from the remote without merging into your current branch.
git fetch origin
Pull
Pull Remote Changes
Fetch and immediately merge remote changes into your current branch.
git pull origin <branch-name>
Pull
Pull with Rebase
Pull remote changes but rebase your local commits on top instead of merging — keeps history clean.
git pull --rebase origin <branch-name>
Delete
Delete Remote Branch
Remove a branch from the remote repository.
git push origin --delete <branch-name>

Tracking Path Changes
Version file moves, renames, and deletions
Move
Rename / Move a File
Move or rename a file and automatically stage the change for the next commit.
git mv <old-path> <new-path>
Delete
Remove a File
Delete a file from the working directory and stage its removal for the next commit.
git rm <filename>
Untrack
Stop Tracking a File
Remove a file from version control but keep it in your local working directory.
git rm --cached <filename>
Log
Log File History
Show the full commit log for a specific file, even across renames.
git log --follow <filename>

Ignoring Patterns
Prevent unintended files from being tracked by Git
.gitignore Syntax
Create a .gitignore file in your repo root. Common patterns:
  • node_modules/ — ignore a whole directory
  • *.log — ignore all .log files
  • .env — ignore a specific file
  • !important.log — un-ignore a file
  • build/ — ignore build output
  • **/__pycache__/ — ignore in any subdirectory
Global
Global .gitignore
Apply a gitignore file across all your repositories on this machine (e.g., OS files, editor configs).
git config --global core.excludesfile ~/.gitignore_global
Check
Show Ignored Files
List all files currently being ignored by Git in the working directory.
git ls-files --others --ignored --exclude-standard
Debug
Why Is a File Ignored?
Find which rule in a .gitignore is causing a specific file to be ignored.
git check-ignore -v <filename>

Rewrite History
Amend, rebase, and reset to reshape commit history
Amend
Amend Last Commit
Modify the most recent commit — change its message or add forgotten staged files.
git commit --amend -m "Updated message"
Rebase
Rebase onto Branch
Replay your current branch's commits on top of another branch, creating a linear history.
git rebase main
Rebase
Interactive Rebase
Interactively squash, reorder, or edit the last N commits. Very powerful for clean PRs.
git rebase -i HEAD~3
Reset
Soft Reset
Move HEAD back N commits but keep all changes staged. Useful for re-committing differently.
git reset --soft HEAD~1
Reset
Mixed Reset (Default)
Move HEAD back and unstage changes, but keep them in the working directory.
git reset HEAD~1
Reset
Hard Reset ⚠️
Discard all commits and changes after the given point. This is destructive!
git reset --hard HEAD~1
Revert
Revert a Commit
Create a new commit that undoes the changes from a specific commit — safe for shared branches.
git revert <commit-hash>
Cherry-pick
Cherry-pick a Commit
Apply the changes from a specific commit onto the current branch.
git cherry-pick <commit-hash>
🗄️
Temporary Commits
Save and restore work-in-progress without committing
Stash
Stash Changes
Temporarily shelve all uncommitted changes so you can work on something else.
git stash
Stash
Stash with Message
Save a stash with a descriptive name to identify it later.
git stash push -m "work in progress"
Stash
Include Untracked Files
Stash both tracked changes and new untracked files.
git stash -u
List
List Stashes
Display all saved stashes in the stash stack.
git stash list
Restore
Apply Latest Stash
Restore the most recently stashed changes without removing it from the stash stack.
git stash apply
Restore
Pop Latest Stash
Apply the most recent stash and remove it from the stash stack.
git stash pop
Restore
Apply Specific Stash
Apply a specific stash entry by its index from git stash list.
git stash apply stash@{2}
Delete
Drop a Stash
Remove a specific stash entry permanently from the stack.
git stash drop stash@{0}
Delete
Clear All Stashes
Remove all stash entries at once. This cannot be undone.
git stash clear
